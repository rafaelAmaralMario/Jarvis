# RAG — Retrieval-Augmented Generation

## Descrição
Implementar pipeline RAG completo: chunking de documentos → embeddings → vector store (ChromaDB) → busca semântica → contexto aumentado para o LLM. Embeddings gerados localmente (NativeProvider ou Ollama). Memória de longo prazo para consultar documentos.

## Critérios de Aceitação
- [ ] Instalar ChromaDB + sentence-transformers
- [ ] Pipeline de chunking (texto → chunks com overlap)
- [ ] Geração de embeddings local (llama-cpp-python ou Ollama)
- [ ] Indexação de documentos no vector store
- [ ] Busca semântica (similaridade de cosseno)
- [ ] Context augmentation no prompt do LLM
- [ ] Tool `query_documents`: pergunta → busca → resposta com fontes
