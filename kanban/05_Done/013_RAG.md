# RAG — Retrieval-Augmented Generation

## Descrição
Implementar pipeline RAG completo: chunking de documentos → embeddings → vector store (ChromaDB) → busca semântica → contexto aumentado para o LLM. Embeddings gerados localmente (NativeProvider ou Ollama). Memória de longo prazo para consultar documentos.

## Critérios de Aceitação
- [x] Instalar ChromaDB (chromadb 1.5.9)
- [x] Pipeline de chunking (`chunk_text`: parágrafos + sentences, config chunk_size/overlap)
- [x] Geração de embeddings via Ollama API (`/api/embed` → `nomic-embed-text`) + fallback hash
- [x] Indexação de documentos no vector store (`index_document`: PDF/DOCX/TXT → chunks → embed → ChromaDB)
- [x] Busca semântica (ChromaDB cosine similarity) + resultado com score de relevância
- [x] Context augmentation (`query_with_context`: busca → formata contexto com fontes)
- [x] Tools: `query_documents`, `index_document`, `rag_list_documents`
