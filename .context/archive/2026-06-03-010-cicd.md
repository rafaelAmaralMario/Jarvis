## Decisao

Criado pipeline CI/CD com GitHub Actions para validação automatizada em push/PR para main.

### Jobs

| Job | Runner | Passos | Descrição |
|-----|--------|--------|-----------|
| **ui** | ubuntu-latest | `npm ci` → `tsc --noEmit` → `vitest run` | Valida TypeScript + testes React |
| **server** | ubuntu-latest | `npm ci` → `npm run build` | Compilação TypeScript do sync server |
| **kernel** | ubuntu-latest | Qt6 install → CMake → build kernel-tests → ctest | Build e testes C++ com Catch2 |
| **docker** | ubuntu-latest | Docker Buildx → build-push-action | Build da imagem sync-server (main only) |

### Detalhes

- **Node 22** com cache de `node_modules` nos jobs UI e Server
- **Qt6 6.7.2** via `jurplel/install-qt-action@v4` com módulos `qtwebengine`, `qtwebsockets`, `qtwebchannel`
- CMake configurado com `-DBUILD_TESTING=ON`; build apenas do target `kernel-tests` (evita linkar WebEngineWidgets para testes)
- Testes C++ executados via `ctest --output-on-failure`
- Docker job executa apenas em push para `main`; imagem não é publicada em registry (apenas build check)
- Dependências Linux: `catch2`, `cmake`, `g++`, `libgl1-mesa-dev`
