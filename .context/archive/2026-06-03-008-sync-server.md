### REST API

```
POST   /api/auth/register       → 201 { user, token }
POST   /api/auth/login          → 200 { user, token }
GET    /api/notes?workspace=... → notes[]
POST   /api/notes               → 201 note
PUT    /api/notes/:id           → 200 note (version check, 409 on conflict)
DELETE /api/notes/:id           → 204 (soft delete)
POST   /api/workspaces          → 201 workspace
GET    /api/workspaces          → workspaces[]
GET    /api/users/me            → user profile
GET    /api/users/search?q=...  → users[]
```

### WebSocket Events

| Direção | Evento | Descrição |
|---------|--------|-----------|
| C→S | subscribe { workspaceId } | Entrar em sala |
| C→S | note:update { noteId, content, version } | Atualizar nota |
| C→S | note:select { noteId } | Notificar seleção |
| C→S | ping | Health check |
| S→C | note:updated { noteId, content, version, updatedBy } | Broadcast |
| S→C | note:deleted { noteId } | Broadcast |
| S→C | note:conflict { noteId, currentVersion, diff } | Conflito |
| S→C | note:selected { noteId, userId } | Broadcast |
| S→C | user:joined / user:left | Presença |
