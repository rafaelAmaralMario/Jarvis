## Resumo

Implementação do `TaskPlanner` — um planejador de tarefas que recebe uma descrição de alto nível, decompõe em passos atômicos usando o LLM, e executa cada passo com verificação, retry automático e relatório consolidado.

## Decisões Técnicas

### Arquitetura
- Nova classe `TaskPlanner` em `backend/jarvis/tool_agent.py` (mesmo arquivo que `ToolAgent`)
- Fluxo: `execute(task)` → `_decompose(task)` → para cada step: `_execute_step(goal, verification)` → `_verify(criteria)` → resultado consolidado
- `_decompose` usa LLM com system prompt específico para gerar plano estruturado em JSON
- `_execute_step` cria um `ToolAgent` novo para cada step, com suporte a `on_token` callback
- Retry automático: até `max_retries_per_step` (padrão 3) em caso de falha de verificação
- `_verify` usa o LLM para checar critérios de verificação (ex: "o arquivo X existe", "o teste Y passa")

### Interface
- Bridge: `taskPlannerExecute(query)` → `TaskPlannerResult` (JSON)
- TypeScript: `TaskPlannerResult`, `TaskPlannerStepResult` interfaces em `types/index.ts`
- Hook: `taskPlannerExecute` em `use-jarvis.ts`
- Mock: bridge mock em `__mocks__/bridge.ts`

### Formato do Plano
```json
{
  "summary": "Brief overview",
  "steps": [
    {
      "goal": "Clear step description",
      "verification": "How to verify success"
    }
  ]
}
```

### Formato do Resultado
```json
{
  "success": true,
  "task": "...",
  "plan_summary": "...",
  "total_steps": 5,
  "completed_steps": 5,
  "successful_steps": 5,
  "results": [
    {"goal": "...", "success": true, "output": "...", "retries": 0}
  ],
  "cancelled": false
}
```
