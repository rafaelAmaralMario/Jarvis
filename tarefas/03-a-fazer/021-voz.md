## Descrição
Interface de voz com o assistente: Speech-to-Text (STT) via whisper.cpp
e Text-to-Speech (TTS) via biblioteca local.

## Especificação Técnica

### 1. Dependências Externas

**whisper.cpp** (STT):
- Repo: https://github.com/ggerganov/whisper.cpp
- Modelo: `ggml-small.bin` (~500MB) ou `ggml-base.bin` (~150MB)
- Integração: API C do whisper.cpp
- Download automático do modelo na primeira execução

**TTS:**
- Opção 1: `espeak-ng` (simples, robótico, já disponível em Linux)
- Opção 2: `piper-tts` (C++, neural, quality melhor)
- Opção 3: Windows SAPI / macOS NSSpeechSynthesizer

### 2. C++ — VoiceManager

**Novo arquivo:** `kernel/include/jarvis/voice/voice_manager.h`
**Novo arquivo:** `kernel/src/voice/voice_manager.cpp`

```cpp
namespace jarvis::voice {

class IVoiceManager {
public:
    virtual ~IVoiceManager() = default;

    // STT
    virtual bool isMicrophoneAvailable() = 0;
    virtual bool startRecording() = 0;
    virtual std::string stopRecording() = 0;     // retorna texto transcrito
    virtual bool isRecording() = 0;

    // TTS
    virtual bool speak(const std::string& text) = 0;   // fala texto
    virtual bool stopSpeaking() = 0;
    virtual bool isSpeaking() = 0;
    virtual void setVoice(const std::string& voice) = 0;

    // Config
    virtual bool isAvailable() = 0;              // STT + TTS configurados?
    virtual std::string getStatus() = 0;         // "ready", "no_microphone", "no_model", "error:..."

    // Eventos
    virtual void setEventCallback(std::function<void(const std::string& event, const std::string& data)> cb) = 0;
};

IVoiceManager* createVoiceManager(const std::string& modelsPath);

}
```

**Implementação STT:**
- Captura de áudio: `QAudioRecorder` ou `QAudioSource` (Qt Multimedia)
- Salva em buffer WAV temporário
- Chama `whisper_full()` do whisper.cpp com o áudio
- Retorna texto transcrito (string UTF-8)

**Implementação TTS:**
- Windows: `SAPI.SpVoice` via COM
- Linux: `espeak-ng` via QProcess
- macOS: `say` command via QProcess
- Fallback para todos os SO: espeak-ng via QProcess

**Download do modelo:**
- Primeira execução: baixa `ggml-base.bin` do HuggingFace
- Progresso emitido via evento `voice-model-download-progress`
- Modelo salvo em `{modelsPath}/whisper-base.bin`

### 4. Bridge handlers

```cpp
bridge.registerHandler("voiceStartRecording", [voiceManager](const QVariantList&) -> QVariant {
    return voiceManager->startRecording();
```

### 5. React — Componentes

**MicrophoneButton.tsx:**
- Botão na StatusBar ou no AiPanel (ao lado do input de chat)
- Estados: idle 🎤, recording 🔴 (pulsando), processing ⏳, error ❌
- Clique inicia/para gravação
- Atalho Ctrl+Shift+M (push-to-talk: segurar para gravar, soltar para transcrever)

**VoiceSettingsPanel.tsx:**
- Integrado na SettingsPage
- Toggles: STT enabled, TTS enabled
- Select: modelo (tiny/base/small), idioma, voz TTS
- Status: "Pronto" / "Modelo não encontrado" / "Microfone indisponível"
- Botão "Download do modelo whisper"

**Integração com AiPanel:**
- Após stopRecording(), texto transcrito aparece no input do chat
- Se TTS habilitado, resposta do assistente é lida em voz automaticamente

### 6. Fluxo de Uso

1. Usuário clica 🎤 na StatusBar
2. VoiceManager.startRecording() → QAudioSource capturando
3. Indicador 🔴 pulsando na interface
4. Usuário fala: "Crie uma nota sobre Python"
5. Usuário clica 🎤 novamente (ou solta Ctrl+Shift+M)
6. VoiceManager.stopRecording() → whisper.cpp transcreve
7. Texto "Crie uma nota sobre Python" aparece no input do chat
8. Usuário envia → AI Engine processa → resposta aparece
9. Se TTS ativo: voiceManager.speak(resposta) → áudio reproduzido
